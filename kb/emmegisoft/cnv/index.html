FP Pro'S CNV File

FP PRO lets the user insert fittings and kits into multijoints, structures and blocks when this happens, a reference to the inserted item (brand, series and code for fittings; group, kit name and description for kits) is introduced into the correspondig file (.BLK for blocks, .STR for structures and .NOD for multijoints). Once this reference is added, FP PRO "disconnects" it from the database, so that any modification made in FP PRO DATABASE to the data inserted in the file (e.g. renaming of the fitting) is not reflected on the multijoints, blocks and structures. To resolve this issue we have defined a way to update the information present in BLK, STR and NOD files.

Activation

This function can be activated in two ways:

  • Through a combination of keys to be entered while in FP PRO:

    CTRL+ALT+SHIFT+C

    FP PRO asks for a file with a CNV extension.

  • From version 6.0/11, using the command line:

    fp_pro.exe --cnv path/to/cnvfile.cnv

Usage

The CNV file has an INI structure, with sections defined by square brackets and keys in each section. The keys in a section can be single (e.g. COPYFROM) or multiple; in which case they must be named with a counter (e.g. FILE1, FILE2 …​). There can also be multiple sections of the same name, multijoints creation (NODEDEF1 …​ NODEDEFn) is a sample thereof. The order in which sections are entered in the CNV file is not important, however, it’s recomended to start with [FILELIST].

Creating a CNV file means defining the action (ACTION key) and which items must be modified: multijoints, structures, blocks, compound profiles (FILE1…​FILEn keys). What has been described up to now is contained in the section [FILELIST] that must be present in any CNV file.

Action Effect

CONVERT

Item modification

REPORT

Report generation (file)

SYNCHRONIZE

Synchronisation of a file with DB

INSERT

Multijoints addition

Having done this, what to change inside the selected files has to be decided: you can act on single profiles, fittings, kits or brands, series and kit groups. The table below makes a summary of the available modifications.

Reference

FILELIST Section

This is the first section that FP PRO reads from the CNV file. It lists the action to be accomplished and the files that must be modified or read.

[FILELIST]
ACTION=<action>
FILE1=<file name STR/BLK/NOD/ADD with complete path>
FILE2=<file name STR/BLK/NOD/ADD with complete path>
...
FILEn=<file name STR/BLK/NOD/ADD with complete path>

ACTION Key

Parameters Purpose

ACTION = CONVERT

(Default) Changes names of fittings, kits, series (profile and fittings), brands, profile names and kit groups.

ACTION = REPORT

Generates report files that contain the list of fittings and kits present in the specified files (no conversion is executed). In particular three text files are created (see description in the reference section)

ACTION = SYNCHRONIZE

Synchronizes fittings and kits present in multijoints, blocks and structures with those present in the database. It erases those that are no more present and updates the description of those that are still present. Only FILE1…​FILEn must be specified and no other section

ACTION = INSERT

adds new multijoints

FILE Key

FILEn = <complete path to file\\filename.ext>

N is a counter tha starts from 1. Any number of FILEn specifiers can be used. The path must be complete (e.g. C:\FP_PRO\NOD\DEMO.NOD). Wildcards are also supported (e.g. C:\FP_PRO\NOD\*.NOD). Permitted extensions are STR, BLK, NOD and ADD.

SERIELIST Section

Substitutes the name of the series in multi-joints, blocks, structures and .add files.

[SERIELIST]
SERIE1 = <old name>,<new name >
...
SERIEn = <old name>,<new name >

DXFLIST Section

Substitutes profiles in multijoints, blocks, structures and add files.

[DXFLIST]
DXF1=<old series|old code>, <new series| new code>
...
DXFn=<old series|old code>, <new series| new code>

KITLIST Section

Substitutes kits in blocks and structures.

[KITLIST]
KIT1 = <old group>|<old name>,<newgroup>|<new name>|<new quantity(optional)>[,<new group>|<nuovo nome>|<new quantity(optional)> , ...]
KIT2 = <old group>|<old name>,<newgroup>|<new name>|<new quantity(optional)>[,<new group>|<nuovo nome>|<new quantity(optional)> , ...]
...
KITn = <old group>|<old name>,<newgroup>|<new name>|<new quantity(optional)>[,<new group>|<nuovo nome>|<new quantity(optional)> , ...

For each line the following information must be specified:

  • original group name and kit name (if the group is "COMMON", leave the field empty)

  • new group name and kit name (if the group is "COMMON", leave the field empty)

  • (Optional) new kit quantity. Put 0 if the original kit must be removed

A kit can be substituted by more than one kit (insert the other kits in a comma-separated list)

FITTINGLIST Section

Substitutes fittings in multijoints, blocks and structures.

[FITTINGLIST]
FITTING1=<old brand>|<old series>|<old code>, <new brand>|<new series>|<new code>|<new quantity (optional)> [, <new brand>|<new series>|<new code>|<new quantity (optional)>, ...]
FITTING2=<old brand>|<old series>|<old code>, <new brand>|<new series>|<new code>|<new quantity (optional)> [, <new brand>|<new series>|<new code>|<new quantity (optional)>, ...]
...
FITTINGn=<old brand>|<old series>|<old code>, <new brand>|<new series>|<new code>|<new quantity (optional)> [, <new brand>|<new series>|<new code>|<new quantity (optional)>, ... ]

For each line the following information must be specified:

  • original fitting brand, series, code

  • new fitting brand, series, code

  • (Optional) new fitting quantity. Put 0 if the original kit must be removed

A fitting can be substituted by more than one fitting (insert the other fittings in a comma-separated list)

FITBRANDLIST Section

Substitutes brands or series from multijoints, structures and blocks. It’s useful if in FP PRO Database an entire brand or series has been renamed.

[FITBRANDLIST]
BRAND1=<old brand|old series>, <new brand|new series>
...
BRANDn=<old brand|old series>, <new brand|new series>

KITGROUPLIST Section

Substitutes kit groups from structures and blocks. It’s useful if in FP PRO Database an entire kit group has been renamed.

[KITGROUPLIST]
GROUP1=<old group>, <new group>
...
GROUPn=<old group>, <new group>

NODEDEF Section

It’s possible to create multijoints: the creatable multijoints are of two kinds, angle or side-cut. Between the two there exists a fundamental difference: angle multijoints can be created only with fittings and without a "template" multijoints, while side-cut multijoints must be created basing on a "template" multijoint.

For "template" multijoint we mean a multijoint of the same type of the one that must be created: the characteristics of the template will be reproduced in the new multijoints, to which the supplied fitting list is appended. If no fitting list is supplied, the one of the template wil lbe appended.

The properties of the template will be "blindly" reproduced on the target multijoint, so extra care must be taken to be sure that the two multijoints are compatible.

[NODEDEF]
COPYFROM=<series1>|<code1>|<[HOT o COLD]>,<series2>|<code2>|<[HOT o COLD]>,<[ANGLE o SIDECUT]>
NODE1=<series1>|<code1>|<[HOT o COLD]>,<series2>|<code2>|<[HOT o COLD]>,<[ANGLE o SIDECUT]>
...
NODEn=<series1>|<code1>|<[HOT o COLD]>,<series2>|<code2>|<[HOT o COLD]>,<[ANGLE o SIDECUT]>
FITTING1=<brand>|<series>|<code>|<quantity>|<[90DEG o NOT90DEG]>
...
FITTINGn=<brand>|<series>|<code>|<quantity>|<[90DEG o NOT90DEG]>
NODE and COPYFROM Key

specifies the template multijoint to be used; only one COPYFROM is allowed in each NODEDEF section. It can be skipped if an angle multijoint has to be created. COPYFROM and NODE have the same format.

  • series1|code1 and series2|code2 are multijoint profiles (no 3-profile multijoint is creatable)

  • HOT or COLD is profile view: it’s important to set it right for side-cut multijoints. Set only one of the two for each profile.

  • ANGLE or SIDECUT correspond to the multijoint type of the same name. Set one of the two for each multijoint.

FITTING Key

  • Brand, series, code

  • Fitting brand, series and code

  • Quantity* number of fittings to be inserted.

  • 90DEG or NOT90DEG

  • The fitting must be used in multijoints with cuts equal to 90 degrees or not. Set only one of the two.

Examples:

Sample CNV File with descriptions

[FILELIST]
ACTION=CONVERT                  (1)
FILE1=C:\fp_pro\NOD\demo.nod
FILE2=C:\fp_pro\NOD\glasses.nod
FILE3=C:\fp_pro\blk\demo\*.blk
FILE4=C:\fp_pro\str\demo\*.str

[KITLIST]
KIT1=G1|K1, G1|K1|2             (2)
KIT2=G2|K2, G1|K3               (3)
KIT3=G3|K3, G5|K4|2, G5|K5|5

[FITTINGLIST]
FITTING1=MM|SS|A1, MM|SS|A2|3   (4)
FITTING2=MM|SS|A3, MMMM|SSSS|A4 (5)

  1. The modification are made on the files of DEMO and GLASSES series, and to all the blocks and structures of the DEMO series.

  2. kit K1 of G1 group gets a | | | quantity of 2; | |

  3. kit K2 of G2 group is substituted by kit K3 of G1| group; with a quantity | | equal to 3; | | | | |

  4. fitting A1 of series SS and | | | brand MM is substituted with | | fitting A2 of the same brand | | | and series

  5. Fitting A3 of series SS and brand MM is substituted with fitting A4 of series SSSS and brand MMMM.

REPORT_NOD.CSV File

  1. Multijoints file name

  2. Multijoin type:

    1. || side-side multijoin

    2. |- side-cut multijoin

    3. cut-cut multijoin

    4. |_ angle multijoin

  3. First profile series

  4. First profile code

  5. First profile use

  6. Second profile series

  7. Second profile code

  8. Second profile use

  9. Central profile series (can be null)

  10. Central profile code (can be null)

  11. String ACCESSORIO (useless, but requested. It means fitting in Italian)

  12. Quantity (number of fittings)

  13. String OPZ if the fitting is optional (can be null)

  14. Reference dimension (as the fitting is inserted in Fp Pro — can be null)

  15. Calculation dimension (as the fitting is inserted in Fp Pro — can be null)

  16. Fitting brand

  17. Fitting code

  18. Fitting description

REPORT_STR.CSV File

  1. STR or BLK file name

  2. String KIT or ACCESSORIO (fitting in Italian)

  3. Quantity (kit or fitting number)

  4. String OPZ if the fitting or kit is optional (can be null)

  5. Reference dimension (as the fitting is inserted in Fp Pro — can be null)

  6. Calculation dimension (as the fitting is inserted in Fp Pro — can be null

  7. Fitting series or kit group

  8. Fitting or kit code

  9. Kit or fitting description
