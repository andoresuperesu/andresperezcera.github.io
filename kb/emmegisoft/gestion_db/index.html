Gestión deDatabase

La base de datos en FP Pro puede instalarse en 2 versiones y gestionarse en varias maneras. Lo importante a saber es lo siguiente:

El programa instala inicialmente todos sus datos base en una base de datos de Access; este es un archivo con extensión ".mdb" y no necesita tener Access instalado en su ordenador. Este tipo de base de datos es bueno si y solo si lo usarán un maximo de 3 usuarios. Por estar basado en un archivo tiene un numero de concurrencia muy bajo.

En los clientes que desean mejorar velocidad o que el numero de usuarios del programa FP Pro supera los 3 anteriormente mencionados, deben usar el motor de base de datos MySQL. Por asuntos de licencia recomendamos usar la versión de comunidad que no tiene restricciones en el uso comercial y no requiere licencias adicionales. La versión actualmente recomendada es la 5.5.8. También necesitara instalar los conectores de .net (versión 6.8.3) y odbc (versines 3.51.19 o 3.51.22).

También es posible utilizar la versión 8 de MariaDB, que es una base de datos creada por el desarrollador original de MySQL cuando esta fué adquirida por Oracle.

Actualización de la base de datos

Existen 2 tipos de actualización de base de datos: Las actualizaciones parciales que se hacen durante los service packs y la actualización completa (creación de DB). La diferencia consiste en que la primera asume que el resto de la estructura de la base de datos se encuentra en correctas condiciones, mientras que la segunda revisa la estructura de cada tabla y la integridad de los datos. Si bien una actualización completa ayuda a solucionar algunos problemas, algunos inconvenientes en generación de metadatos relevantes para el programa requiere la intervención directa de soporte. No ejecute la una ni la otra si no entiende lo que hace. Siempre realice un respaldo de los datos de la base de datos y archivos de trabajo antes de proceder.

Actualización de service pack

Este tipo de actualización se realiza cada vez que se instala un service pack. Sin embargo, cuando las bases de datos son compartidas en red en modo cliente servidor, es preferible instalar el service pack con la ruta base por default del programa y luego actualizar la base de datos. Esto se realiza en la consola de servicio (service console). Los pasos son los siguientes:

  1. Abra la consola de servicio (Service Console).

  2. EN la parte superior izquierda está el botón Gestionar Base de datos. Arriba de este botón hay 2 Access y MySQL; bien sea el uno o el otro debe estar en verde su indicador.

  3. Haga click en gestionar base de datos.

  4. En la nueva ventana haga click en probar conexión.

  5. Si la conectó exitosamente a la base de datos, presione el botón actualizar db. Es posible que esté en italiano "Aggiorna versione DB".

Actualización completa

La única manera de obtener una actualización completa de la base de datos (especialmente necesarios al saltar varios service packs), es por medio de la ejecución del siguiente comando. Tenga en cuenta que solo debe ejecutarse una vez cuando los archivos son compartidos en RED.

    C:\FP_PRO\upgradedb.exe 100 /d
